Test Your DKU AI Token

Quick verification for students and faculty before building or coding with DKU AI services.
在开始使用 DKU AI 服务进行开发或测试前，可先用此页面快速验证 token 是否可用。

1Copy your token复制你的 token

Open your token manager and copy the token that starts with sk-.
打开 token manager，复制以 sk- 开头的 token。

Important / 重要： keep your token private. Do not share it with others and do not post it in screenshots, chat messages, or public code repositories.
请妥善保管你的 token，不要分享给他人，也不要出现在截图、聊天记录或公开代码仓库中。

2Open Terminal打开 Terminal

Open Terminal on macOS, Windows Terminal, or another command-line tool.
打开 macOS 的 Terminal、Windows Terminal，或其他命令行工具。

You will paste a single curl command to test whether your token can access the DKU AI service.
你将粘贴一条 curl 命令，用于测试 token 是否可以访问 DKU AI 服务。

3Run the test command运行测试命令

Replace sk-xxxxxxxx with your own token, then run the command below:
将 sk-xxxxxxxx 替换成你自己的 token，然后运行下面的命令：

curl -sS https://aichat.dukekunshan.edu.cn:3443/v1/chat/completions \
  -H "Authorization: Bearer sk-xxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5-mini",
    "messages": [
      {"role": "user", "content": "What day is it today and what happened in history on this date?"}
    ],
    "max_tokens": 10000
  }'

4Check the result检查返回结果

If your token works, you should see a JSON response in Terminal. That means your token is valid and can access the model.
如果 token 可用，你会在 Terminal 中看到 JSON 返回结果。这表示你的 token 有效，并且可以访问模型。

  • Success / 成功： you receive a structured JSON response with model output.
    你会收到包含模型输出的 JSON 结果。
  • Unauthorized / invalid token： check whether you copied the full token correctly.
    未授权 / token 无效： 请检查是否完整、正确地复制了 token。
  • No response / connection issue： check network access and try again.
    无返回 / 连接异常： 请检查网络连接后重试。
